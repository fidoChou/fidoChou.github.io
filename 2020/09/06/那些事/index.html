Redis 那些事

2020.09.06 Jojo  热度 ℃

「纸上得来终觉浅，绝知此事要躬行」

2008 年，一家意大利创业公司嫌弃 MySQL 的性能，所以就自己搞了套数据库：Redis。这个个人开发的数据库究竟有哪些特性吸引着开发者？

特性

存储结构

Redis 的全称是 Remote Dictionary Server（远程字典服务器）

Redis 的键值除了是字符串还可以是其他数据类型：

  1. 字符串
  2. 散列
  3. 列表
  4. 集合
  5. 有序集合

内容存储与持久化

所有的数据都存储在内存中，内存的读写速度快于硬盘，一秒内可读写超过十万个键值。

功能丰富

键值的 TTL 、单线程模型、可实现队列

简单稳定

入门

字符串类型

字符串类型是 Redis 中最基本的类型，它能存储任何形式的字符串，包括二进制数据。是其他 4 种数据类型的基础，一个字符串类型键最大哦的容量是 512 MB。

SET key value
GET key

是最简单的操作

散列类型

散列即 hash，其存储了字段 field 和 字段值的映射，「但字段值只能是字符串」，不支持其他数据类型。所以散列类型不能嵌套其他数据类型。

key: 对象类型 + id
field：属性
value：字段值

HSET key field value
HGET key field

列表类型

Redis 列表类型内部使用双向链表实现，常用操作是两列表两端添加元素，或者获取列表片段。列表类型适合用来记录日志

LPUSH key value
RPUSH key value

集合类型

集合：不保证顺序、唯一

SADD key member
SREM key member

有序集合

ZADD key score memeber
